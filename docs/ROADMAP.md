## 📋 다음 작업 목록

### Phase 1: 핵심 UX 개선 (우선순위: HIGH)

#### 1. 영양 화면 날짜 선택 달력

**목표**: 특정 날짜의 영양 기록 조회

**현재 상태**:

- 오늘 ◀ ▶ 어제/내일로만 이동 가능
- 한 달 전 기록 보기 어려움

**구현 방법**:

```javascript
// NutritionScreen.js
- 날짜 텍스트 영역 탭 → 달력 팝업
- react-native-calendars 라이브러리 활용
- 원하는 월/일 선택 → 해당 날짜 영양 기록 조회
```

**설치**:

```bash
npx expo install react-native-calendars
```

**예상 소요 시간**: 2-3시간

---

#### 2. 레시피 추천 고도화

**목표**: 냉장고 식재료로 만들 수 있는 레시피 정확도 향상

**개선 항목**:

- [ ] 부분 재료 매칭 (모든 재료가 없어도 추천)
- [ ] 난이도별 필터링
- [ ] 조리 시간별 정렬
- [ ] 영양 목표에 맞는 추천 (다이어트/벌크업)

**백엔드 작업**:

```java
// AppFeatureService.recommend()
- 점수 계산 알고리즘 개선
- 가중치 조정 (완벽 매칭 > 부분 매칭)
- SQL 쿼리 최적화
```

**예상 소요 시간**: 3-4시간

---

#### 3. 영양 목표 추적 대시보드

**목표**: 일일/주간/월간 영양 섭취 추적 시각화

**추가 기능**:

- [ ] 그래프 (칼로리, 단백질, 지방, 탄수화물)
- [ ] 권장 섭취량 대비 진행률
- [ ] 주간 요약 (평균 칼로리, 목표 달성률)
- [ ] 월별 통계

**FrontEnd**:

```javascript
// NutritionScreen.js 확장
- 차트 라이브러리: react-native-svg-charts
- 날짜 범위 선택 (주간/월간)
```

**예상 소요 시간**: 4-5시간

### Phase 2: 기능 확장 (우선순위: MEDIUM)

#### 4. 바코드 스캔 기반 장보기

**목표**: 마트에서 제품 바코드를 스캔하여 장보기 항목으로 추가

**구현 방법**:

```javascript
// ShoppingScreen.js
- 카메라 권한 요청
- expo-camera로 바코드 인식
- OpenFoodFacts API로 제품명 조회
- 자동 장보기 항목 추가
```

**설치**:

```bash
npx expo install expo-camera expo-barcode-scanner
```

**예상 소요 시간**: 3-4시간

## 💡 개발 팁

### 의존성 설치 시

```bash
npx expo install <package-name>
# npm install 대신 expo install 사용 (expo 호환성 확보)
```

### 테스트

```bash
# 전체 테스트
npm test

# 특정 파일만 테스트
npm test -- NutritionScreen.test.js
```

### 커밋 메시지 컨벤션

```
[FEAT] 새 기능 추가
[FIX] 버그 수정
[DOCS] 문서 업데이트
[REFACTOR] 코드 리팩토링
[PERF] 성능 개선
```
