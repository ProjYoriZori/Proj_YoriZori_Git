## 📋 개요

YoriZori 서비스의 전체 사용자 여정을 커버하는 QA 테스트 케이스입니다.

**테스트 범위**:

- 앱 초기화 및 렌더링
- 회원가입 / 로그인
- 사용자 정보 입력
- 메인 기능 (레시피, 냉장고, 추천, 영양)
- 마이페이지 및 로그아웃

**우선순위**:

- **P1**: 핵심 기능 (배포 필수)
- **P2**: 주요 기능 (릴리즈 전 권장)
- **P3**: 부가 기능 (고도화 단계)

## 🧪 테스트 시나리오

### 시나리오 01: 앱 실행

| 테스트 ID | 시나리오             | 목적                | 예상 결과                           | 우선순위 |
| --------- | -------------------- | ------------------- | ----------------------------------- | -------- |
| TC-001    | 앱 최초 진입         | 홈 화면 정상 렌더링 | 2초 이내 렌더링, 탭 네비게이션 표시 | P1       |
| TC-002    | 공공 API 데이터 로딩 | 레시피 데이터 노출  | 1건 이상 레시피 표시                | P1       |
| TC-003    | API 서버 다운        | 오류 메시지 표시    | "서비스 문제" 메시지 표시           | P1       |
| TC-004    | 네트워크 없음        | 오프라인 감지       | "네트워크 연결 확인" 메시지         | P2       |
| TC-005    | 로딩 속도            | 3초 이내 렌더링     | FCP 3초 이내                        | P2       |

### 시나리오 02: 회원가입 / 로그인

| 테스트 ID | 시나리오          | 목적                  | 예상 결과                          | 우선순위 |
| --------- | ----------------- | --------------------- | ---------------------------------- | -------- |
| TC-006    | 정상 회원가입     | 신규 사용자 가입 성공 | 프로필 입력 화면으로 이동          | P1       |
| TC-007    | 중복 이메일       | 오류 감지             | "이미 사용 중인 이메일" 메시지     | P1       |
| TC-008    | 이메일 형식 오류  | 유효성 검사           | "올바른 이메일 형식" 메시지        | P1       |
| TC-009    | 비밀번호 8자 미만 | 최소 길이 검증        | "8자 이상" 메시지                  | P1       |
| TC-011    | 정상 로그인       | 기존 사용자 로그인    | 홈 화면 진입                       | P1       |
| TC-012    | 비밀번호 틀림     | 오류 감지             | "이메일 또는 비밀번호 확인" 메시지 | P1       |

### 시나리오 03: 사용자 정보 입력

| 테스트 ID | 시나리오            | 목적             | 예상 결과                  | 우선순위 |
| --------- | ------------------- | ---------------- | -------------------------- | -------- |
| TC-015    | 신체 정보 저장      | 프로필 정보 입력 | "저장되었습니다" 메시지    | P1       |
| TC-016    | 나이 최솟값 (1세)   | 경계값 테스트    | 저장 성공                  | P2       |
| TC-017    | 나이 최댓값 (120세) | 경계값 테스트    | 저장 성공 또는 유효성 에러 | P2       |

### 시나리오 04-14: 주요 기능 테스트

- **레시피 검색**: 키워드 검색, 필터링, 상세 조회
- **냉장고 관리**: 식재료 추가/삭제, 기피 재료 설정
- **추천 기능**: 냉장고 기반 레시피 매칭
- **장보기**: 부족 재료 감지, 목록 생성
- **영양 로깅**: 음식 기록, 일일 요약 조회
- **조리 타이머**: 타이머 설정 및 알람
- **마이페이지**: 프로필 수정, 로그아웃

## 🎯 테스트 전략

### 테스트 환경

| 항목     | 값                                     |
| -------- | -------------------------------------- |
| OS       | Windows 11, macOS                      |
| 브라우저 | Chrome, Safari, Edge 최신              |
| 해상도   | 1920×1080 (데스크톱), 390×844 (모바일) |
| 백엔드   | Spring Boot, MySQL 8.x                 |
| 데이터   | 공공데이터 식약처 레시피 DB            |

### 테스트 원칙

1. **카드 경로**: 사용자가 처음 진입하는 순간부터 로그아웃까지 전체 흐름
2. **경계값 테스트**: 최소/최대값, 특수 문자, 빈 필드
3. **예외 케이스**: 서버 다운, 네트워크 오류, 중복 데이터
4. **성능 테스트**: 로딩 속도, API 응답 시간
