# 요리조리 (YoriZori) 프로젝트 컨셉 기획안

---

## 1. 대상 및 목표

### 대상 (주 사용자)

| 대상 | 설명 |
|---|---|
| 자취생 / 1인 가구 | 냉장고 재료를 효율적으로 소비하고 싶은 사용자 |
| 건강 관리 관심자 | 칼로리·영양소를 일별로 추적하고 싶은 사용자 |
| 요리 초보자 | 가진 재료로 뭘 만들 수 있는지 모르는 사용자 |

### 목표

- **식재료 낭비 방지**: 보유 재료 기반 레시피 추천으로 냉장고 속 재료를 남기지 않음
- **건강 관리**: 섭취 음식의 영양 정보를 일별 기록하고 목표 대비 현황 파악
- **편의성 향상**: 레시피 탐색 → 조리 → 장보기까지 하나의 앱에서 완결

---

## 2. 사용된 공공 데이터

### 식품의약품안전처 식품영양성분 레시피 공공데이터

| 항목 | 내용 |
|---|---|
| **데이터 링크** | https://www.foodsafetykorea.go.kr/api/openApiInfo.do (서비스 ID: `COOKRCP01`) |
| **제공 기관** | 식품의약품안전처 (식약처) |
| **데이터 포맷** | XML / JSON |
| **주요 필드** | 레시피명, 조리 유형, 재료 목록, 조리 단계별 설명, 단계별 이미지, 열량·단백질·지방·탄수화물 등 영양정보 |
| **데이터 크기** | 약 1,000건 이상 (식약처 제공 표준 레시피; COOKRCP01 서비스 기준) |
| **활용 방식** | 서버 최초 실행 시 OpenAPI를 호출하여 `recipes`, `recipe_ingredients`, `recipe_steps` 테이블에 수집·저장 |

## 3. DB 구조

### ER 다이어그램

```
┌─────────────┐       ┌──────────────────┐       ┌────────────────┐
│    users    │       │ recipe_ingredients│       │   ingredients  │
│─────────────│       │──────────────────│       │────────────────│
│ id (PK)     │       │ id (PK)          │       │ id (PK)        │
│ email       │       │ recipe_id (FK)   │       │ name           │
│ name        │       │ ingredient_id(FK)│       │ category       │
│ password_hash│      │ quantity         │       │ image_url      │
│ created_at  │       │ unit             │       └──────┬─────────┘
└──────┬──────┘       └──────────────────┘              │
       │                                                 │
       ├──[1:N]──► pantry_items ──[N:1]──────────────────┤
       │            (냉장고 재료)                         │
       │                                                 │
       ├──[1:N]──► avoid_ingredients ──[N:1]─────────────┤
       │            (기피 재료)                           │
       │                                                 │
       ├──[1:N]──► shopping_items ──[N:1]────────────────┘
       │            (장보기 목록)
       │
       ├──[1:N]──► nutrition_logs ──[N:1]──► recipes
       │            (영양 기록)                │
       │                                      ├──[1:N]──► recipe_steps
       ├──[1:N]──► favorites ──[N:1]──────────┘            (조리 단계)
       │            (즐겨찾기)
       │
       └──[1:N]──► custom_foods
                    (커스텀 음식)

ingredients ──[1:N]──► seasonal_ingredients
                        (제철 식재료)
```

### 주요 테이블 요약 (12개)

| 테이블 | 설명 |
|---|---|
| `users` | 사용자 계정 (이메일, 비밀번호 해시) |
| `recipes` | 레시피 (이름, 영양정보 JSON, 난이도, 조리시간) |
| `ingredients` | 식재료 마스터 (이름, 카테고리) |
| `recipe_ingredients` | 레시피 ↔ 재료 N:N 매핑 |
| `recipe_steps` | 조리 단계 (순서, 설명, 이미지) |
| `pantry_items` | 사용자 냉장고 (재료, 수량, 유효기간) |
| `avoid_ingredients` | 기피/알레르기 재료 |
| `nutrition_logs` | 일별 영양 섭취 기록 |
| `shopping_items` | 장보기 목록 (구매 여부 포함) |
| `favorites` | 즐겨찾기 레시피 |
| `seasonal_ingredients` | 제철 식재료 (월별) |
| `custom_foods` | 사용자 정의 음식 (직접 영양 입력) |

## 4. 각 기능별 상세 내용

### 기능별 진행률

| 기능명 | 진행률 | 화면 |
|---|---|---|
| 회원가입 / 로그인 | 100% | AuthScreen |
| 홈 (레시피 추천) | 100% | HomeScreen |
| 레시피 검색 | 100% | RecipesScreen |
| 레시피 상세 + 조리 타이머 | 100% | RecipeDetailScreen |
| 냉장고 관리 | 100% | FridgeScreen |
| 영양 로깅 (OCR 포함) | 100% | NutritionScreen |
| 장보기 목록 | 100% | ShoppingScreen |
| 마이페이지 | 100% | MyPageScreen |
| 영양 달력 (날짜 선택) | 0% | - |
| 영양 대시보드 (차트) | 0% | - |

---

### 기능 1 — 레시피 검색 및 추천

**사용 공공데이터**: 식약처 COOKRCP01 → `recipes`, `recipe_ingredients`, `ingredients`

**데이터 가공 방법**:
1. 식약처 OpenAPI XML 응답 파싱 (`FOOD_API_KEY` 사용)
2. 재료 문자열 토큰화 → `ingredients` 테이블 정규화 (동의어 처리: 계란↔달걀, 대파→파)
3. 검색어와 재료명을 공백 제거 후 비교, 짧은 토큰 부분 매칭 제한
4. 냉장고 재료로 만들 수 있는 레시피를 점수 계산 → 재료 미보유 시 랜덤/최신 fallback

**화면 결과**: 홈 화면에서 추천 레시피 카드, 검색 탭에서 키워드·재료명 검색 결과 목록 표시

---

### 기능 2 — 냉장고 관리 (Pantry)

**사용 공공데이터**: `ingredients` 마스터 테이블 (식약처 데이터에서 추출된 재료 목록)

**데이터 가공 방법**:
- 사용자가 재료를 선택하면 `pantry_items`에 수량·유효기간 저장
- 카테고리별 이모지 분류 (야채, 육류, 유제품 등)
- 기피 재료 등록 시 `avoid_ingredients` 테이블에 추가 → 레시피 추천에서 자동 제외

**화면 결과**: 카테고리별 탭으로 구분된 냉장고 재료 목록, 재료 추가/삭제/수량 수정 UI

---

### 기능 3 — 영양 로깅 (OCR 포함)

**사용 공공데이터**: 식약처 레시피의 `nutrition_info` JSON (칼로리, 단백질, 지방, 탄수화물)

**데이터 가공 방법**:
1. 레시피 선택 → 인분(serving_size) 입력 → 비례 계산하여 `nutrition_logs` 저장
2. **OCR 기능**: Google Vision API로 영양성분표 사진 촬영 → 수치 자동 인식 → `custom_foods`에 저장
3. 일별 합산 쿼리로 총 칼로리·영양소 집계

**화면 결과**: 날짜별 섭취 기록 목록, 일일 총 영양소 요약 카드 (칼로리/단백질/지방/탄수화물)

---

### 기능 4 — 장보기 목록

**사용 공공데이터**: `recipe_ingredients` (레시피별 필요 재료 목록)

**데이터 가공 방법**:
1. 추천 레시피의 재료 목록 조회
2. 냉장고(`pantry_items`)에 없는 재료만 필터링
3. 기피 재료(`avoid_ingredients`) 제외 후 `shopping_items`에 자동 추가
4. 구매 완료 체크 시 `is_purchased = TRUE`, `purchased_at` 기록

**화면 결과**: 장보기 항목 리스트, 체크박스로 구매 완료 표시, 항목별 수량 조정

---

### 기능 5 — 레시피 상세 및 조리 타이머

**사용 공공데이터**: `recipe_steps` (단계별 조리 지침·이미지)

**데이터 가공 방법**: 식약처 OpenAPI의 `MANUAL` 필드를 단계별로 파싱 → `recipe_steps` 테이블에 `step_order` 순으로 저장

**화면 결과**: 단계별 스와이프 조리 가이드, MM:SS 타이머 (테두리 스냅 애니메이션, 색상 선택), 완료 시 벨 소리 + 진동 알람

---

### 기능 6 — 마이페이지

**주요 기능**: 프로필 수정, 로그아웃, 회원 탈퇴 (Soft Delete)

**화면 결과**: 사용자 이름·이메일 표시, 즐겨찾기 레시피 목록, 설정 메뉴

## 5. 데모 시나리오

### 시나리오: "오늘 집에 있는 재료로 저녁 해결하기"

```
[1단계] 앱 실행 및 로그인
  └→ AuthScreen: 이메일/비밀번호로 로그인

[2단계] 냉장고 재료 확인 및 추가
  └→ FridgeScreen: 달걀, 대파, 두부 등록 (카테고리별 이모지 표시)

[3단계] 홈 화면 레시피 추천 확인
  └→ HomeScreen: "달걀 + 두부"로 만들 수 있는 추천 레시피 목록 표시
               → "두부달걀찜" 선택

[4단계] 레시피 상세 조회 및 조리
  └→ RecipeDetailScreen: 재료 목록, 단계별 조리법 확인
                       → 조리 타이머 시작 (5분 설정, 완료 시 알람)

[5단계] 영양 섭취 기록
  └→ NutritionScreen: "두부달걀찜" 1인분 기록
                    → 오늘 총 칼로리 / 단백질 / 지방 / 탄수화물 확인

[6단계] 부족 재료 장보기 목록 생성
  └→ ShoppingScreen: 냉장고에 없는 재료 자동 감지 → 장보기 목록 생성
                   → 마트에서 구매 후 체크 완료
```
